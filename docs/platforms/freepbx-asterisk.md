# FreePBX + Asterisk

The PBX. Image: `escomputers/freepbx:17` (FreePBX 17, Asterisk 21, MariaDB 10.11). Containerized via the upstream `escomputers/freepbx-docker` compose, with subnet relocated to `172.30.0.0/16`.

## Where things live

| Path on host | Purpose |
| --- | --- |
| `/srv/docker/freepbx/docker-compose.yaml` | Stack definition (freepbx + db) |
| `/srv/docker/freepbx/run.sh` | Upstream's iptables RTP NAT helper |
| `/srv/docker/freepbx/{mysql_root_password,freepbxuser_password,sasl_passwd}.txt` | Docker secrets |
| `/srv/docker/freepbx/.credentials.txt` | Mirror of MariaDB creds for humans |
| `/srv/docker/freepbx/asterisk/.ari-credentials.txt` | ARI user (`ava`) + password |
| `/srv/docker/freepbx/dialplan/extensions_custom.conf` | Authoritative custom dialplan |
| `/srv/docker/freepbx/scripts/n8n-call-logger.sh` | Hangup-handler bash script |
| `/srv/docker/freepbx/scripts/n8n-recording-uploader.sh` | MixMonitor close-script |

Inside the container the dialplan + scripts live at `/etc/asterisk/extensions_custom.conf` and `/etc/asterisk/scripts/`.

## Critical configuration

### ARI (HTTP API for AVA)

| Setting | Value |
| --- | --- |
| Bind | `0.0.0.0:8088` |
| Scheme | `http` |
| User | `ava` |
| Password | in `/srv/docker/freepbx/asterisk/.ari-credentials.txt` |
| Stasis app name | `asterisk-ai-voice-agent` |

ARI user lives in `/etc/asterisk/ari_custom.conf` (included from `ari.conf`).

### Custom dialplan (volume-mounted from `/srv/docker/freepbx/dialplan/extensions_custom.conf`)

Active contexts:

| Context | Purpose |
| --- | --- |
| `freepbx-n8n-logger` | Hangup-handler subroutine — runs `n8n-call-logger.sh` on every hangup |
| `macro-dialout-one-predial-hook` | Attaches the hangup handler to outbound dials (FreePBX 17 hook name) |
| `macro-dial-ringall-predial-hook` | Same, for ringall groups |
| `macro-dial-hunt-predial-hook` | Same, for hunt groups |
| `from-ai-agent` | Hands the call off to AVA via `Stasis(asterisk-ai-voice-agent)` |
| `from-internal-custom` → `*99` | Feature code for AVA |
| `from-internal-custom` → `*98` | Feature code for recording-only test (no AI) |
| `n8n-test-caller`, `n8n-ai-test-caller` | Synthetic originator dialplans for CLI testing |

### Extension `1001`

PJSIP extension. Provisioned via `fwconsole bulkimport`.
Secret: `t3st-1001-secret`. Direct media: **`no`** (forced via `sip` table for MixMonitor capture).

## Modifications applied to FreePBX-managed files

These files are auto-regenerated by `fwconsole reload`. Be aware:

| File | Change | Reason |
| --- | --- | --- |
| `/etc/asterisk/http_additional.conf` | `enabled=yes`, `bindaddr=0.0.0.0` | Open ARI on `:8088`. Reverted by `fwconsole reload` — re-apply if needed. |
| `/etc/asterisk/asterisk.conf` | `live_dangerously = yes` | Allow `System()` from dialplan reached via originate (used by hangup script). |
| MariaDB `sip` table for ext 1001 | `direct_media=no` | Without this, RTP goes peer-to-peer and MixMonitor records nothing. |

## Common operations

```bash
# Reload dialplan after editing extensions_custom.conf
docker exec freepbx-freepbx-1 asterisk -rx 'dialplan reload'

# Restart Asterisk (loses live calls)
docker exec freepbx-freepbx-1 asterisk -rx 'core restart now'

# Asterisk console
docker exec -it freepbx-freepbx-1 asterisk -r

# Tail Asterisk full log
docker exec freepbx-freepbx-1 tail -f /var/log/asterisk/full

# See registered SIP endpoints
docker exec freepbx-freepbx-1 asterisk -rx 'pjsip show endpoints'

# Recreate after iptables rules need re-applying
cd /srv/docker/freepbx && bash run.sh --rtp 10000-20000
```

## Gotchas (in our setup)

- **`fwconsole reload` blows away `http_additional.conf` overrides** — `enabled=yes` flips back to `no`. Re-apply after every reload.
- **`chan_local.so` fails to load** at startup (missing from this image), but `Local` channel driver is registered anyway via builtin. Noise, not a blocker.
- **`fwconsole bulkimport` exits with code 2 silently** even on success — verify via DB or `pjsip show endpoint`.
